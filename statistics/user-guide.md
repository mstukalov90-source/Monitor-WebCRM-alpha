# Статистика — руководство пользователя

## Зачем нужна статистика

Система автоматически считает значимые действия сотрудников: обследования в поле, работу с разрытиями, камеральный анализ, закрытие задач. Данные помогают оценить нагрузку и результат работы команды за выбранный период.

Статистика **не требует отдельных действий** — события фиксируются при обычной работе в WebCRM, QGIS или мобильном приложении.

## Кто что видит

| Роль | Что доступно |
|------|----------------|
| **field** (полевой) | Только свои 4 показателя |
| **office** (офис) | Только свои офисные действия (таблица) |
| **manager**, **admin** | Все сотрудники; фильтры по роли, человеку, типу объекта |

## Как открыть

1. Войти в WebCRM.
2. Нажать **«Статистика»** (стартовый экран или шапка).
3. Указать период **С** / **По** и нажать **«Показать»**.

По умолчанию показываются последние 30 дней.

## Полевые сотрудники — 4 показателя

Для каждого полевого сотрудника за период считается:

| Показатель | Когда засчитывается |
|------------|---------------------|
| **Обследование камеральной задачи** | Полевой выполнил обследование задачи, отправленной из офиса (есть отчёт в мобильном приложении и задача снята с «В поле») |
| **Отсутствие разрытия по задаче** | Полевой зафиксировал, что разрытия нет (отчёт + перевод задачи в «разрытие отсутствует») |
| **Обнаружение разрытия в поле** | Полевой обнаружил новое разрытие (отчёт + создание полевой задачи) |
| **Закрытие заказа** | Полевой исполнитель закрыл площадной заказ (статус заказа → «Завершён») |

Руководитель видит таблицу по всем полевым; полевой сотрудник — карточки со своими цифрами.

## Офис — 6 типов действий

Офисная статистика показывается **таблицей**: сотрудник → тип объекта (задача / заказ) → действие → количество.

| Действие | Когда засчитывается |
|----------|---------------------|
| **Анализ начат** | Сотрудник начал камеральный анализ площадного заказа |
| **Анализ завершён** | Анализ заказа завершён |
| **Разрытие отсутствует** | Задача переведена в статус «разрытие отсутствует» из офиса (WebCRM или QGIS) |
| **Создано камеральных задач** | При анализе создана новая задача камеральным методом |
| **Закрыто легально** | Задача закрыта как легальная |
| **Закрыто нелегально** | Задача закрыта как нелегальная |

## Что не попадает в статистику

- Назначение исполнителя, смена вкладок, правка атрибутов без смены статуса.
- Отправка задачи «В поле» (это офисное действие подготовки, не отдельная метрика v2).
- Повторное **то же** действие по **той же** задаче или заказу — второй раз не увеличит счётчик (одна задача = одно событие каждого типа).

## Частые вопросы

**Почему у меня ноль, хотя я работал?**

- Действие могло быть **до** выбранного периода — расширьте даты.
- Для офиса: операция могла выполняться **без указания пользователя** в старых версиях клиента (до конца июня 2026) — такие строки не попали в stat.
- Полевые метрики «с отчётом» требуют синхронизации мобильного приложения с сервером.

**QGIS и WebCRM — одна статистика?**

Да. Оба клиента пишут в одну базу; учёт идёт по факту изменения данных, а не по программе.

**Manager и admin — в чём разница для статистики?**

Нет разницы: оба видят статистику всего персонала с фильтрами.

## Связь с вашей работой (кратко)

```text
Полевой:  мобильное приложение → отчёт + смена статуса задачи / заказа
Офис:     WebCRM или QGIS → анализ заказа, clear / legal / illegal, камеральные задачи
```
